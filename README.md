# Passive-Reconnaissance-Notes
What is Passive Reconnaissance?  Passive reconnaissance is the process of gathering information about a target without directly interacting with it. It helps in understanding the target’s infrastructure while remaining undetected.
🎯 Objective
Collect publicly available information
Identify domain details and DNS records
Map the target’s digital footprint
Avoid detection during the reconnaissance phase
🛠️ Tools Covered
1. WHOIS
Used to retrieve domain registration details
Provides:
Domain owner information
Registration & expiry dates
Registrar details
Helps in identifying ownership and history of a domain
2. NSLOOKUP
Queries DNS to obtain domain-related information
Common uses:
Find IP address of a domain
Retrieve DNS records (A, MX, etc.)
Useful for quick DNS checks
3. DIG (Domain Information Groper)
Advanced DNS query tool
Provides detailed DNS information
Common use cases:
Query specific record types (A, MX, TXT)
Perform DNS troubleshooting
More powerful and flexible than nslookup
📊 Key DNS Record Types
A Record → Maps domain to IPv4 address
MX Record → Mail servers
TXT Record → Verification & security info
NS Record → Name servers
⚡ Importance in Cybersecurity
First phase of penetration testing
Helps identify attack surface
Enables informed decision-making before active testing
Reduces risk of detection
🧠 Key Takeaways
Passive recon = stealthy information gathering
Public data can reveal critical insights
Strong recon = better exploitation strategy
Tools like WHOIS, NSLOOKUP, DIG are foundational
🚀 Conclusion

Passive reconnaissance is a crucial step in ethical hacking that lays the groundwork for deeper security assessments. Mastering these tools builds a strong base for advanced penetration testing techniques.
